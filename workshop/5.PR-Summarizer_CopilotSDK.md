# Exercise 5: PR Summarizer with Copilot SDK

> **Time:** ~40 minutes
> **Standalone:** Requires Exercise 4 (a merged self-healing PR to analyse). Also needs `OPENAI_API_KEY`.

## Goal

Build a Python agent that fetches PR data from GitHub, sends it to the OpenAI API for analysis, and posts the AI summary as a PR comment. Then automate the agent with a GitHub Actions workflow.

---

## Steps

**1.** Create the agent package structure:

```powershell
mkdir agents\pr_summarizer
New-Item agents\__init__.py -ItemType File
New-Item agents\pr_summarizer\__init__.py -ItemType File
```

**2.** Create `agents\pr_summarizer\requirements-agent.txt`:

```text
PyGithub>=2.1.1
openai>=1.0.0
python-dotenv>=1.0.0
pydantic-settings>=2.0.0
rich>=13.0.0
```

Install:

```powershell
pip install -r agents\pr_summarizer\requirements-agent.txt
```

**3.** Create `agents\pr_summarizer\.env` (never commit this):

```env
GITHUB_TOKEN=ghp_your_pat_here
GITHUB_REPOSITORY=CanarysPlayground/Help_Desk_Application
OPENAI_API_KEY=your_openai_key_here
COPILOT_MODEL=gpt-4
```

Add to `.gitignore`:

```powershell
Add-Content .gitignore "`nagents/**/.env"
```

**4.** Use Copilot CLI to generate each module. Open `gh copilot chat` and run each prompt, saving the output to the specified file:

**4.1** `agents/pr_summarizer/config.py`
```
Create a Pydantic Settings class loading GITHUB_TOKEN, GITHUB_REPOSITORY, OPENAI_API_KEY, COPILOT_MODEL from environment variables, reading from .env in the same directory.
```

**4.2** `agents/pr_summarizer/github_integration.py`
```
Using PyGithub, create get_pr_details(repo, pr_number) returning title, body, author, changed files with diffs, and labels. Create post_pr_comment(repo, pr_number, body) to post a comment. Accept a Settings object for auth.
```

**4.3** `agents/pr_summarizer/copilot_analyzer.py`
```
Using the openai client, create analyze_pr(pr_details) that builds a prompt and returns a structured Markdown review: summary, key changes, risks, testing recommendations, and approve/request-changes recommendation.
```

**4.4** `agents/pr_summarizer/agent.py`
```
Main entry point: argparse --pr argument, load settings, fetch PR, analyse, post comment. Use rich for terminal output. Include __main__ block.
```

**5.** Test locally against the self-healing PR:

```powershell
.\.venv\Scripts\Activate.ps1
python -m agents.pr_summarizer.agent --pr <self-healing-pr-number>
```

Open the PR in the browser and confirm the AI comment appears:

```powershell
gh pr view <pr-number> --web
```

**6.** Generate the GitHub Actions workflow. In Copilot chat:

```
Generate .github/workflows/copilot-pr-summarizer.yml:
- Trigger: pull_request opened/synchronize/reopened
- Python 3.11, install agents/pr_summarizer/requirements-agent.txt
- Run: python -m agents.pr_summarizer.agent --pr ${{ github.event.pull_request.number }}
- Secrets: GITHUB_TOKEN (built-in), OPENAI_API_KEY
```

**7.** Add the secret and push:

```powershell
gh secret set OPENAI_API_KEY
git add agents\ .github\workflows\copilot-pr-summarizer.yml
git commit -m "feat: Add Copilot SDK PR summarizer"
git push origin $(git branch --show-current)
```

**8.** Create a test PR to confirm end-to-end automation:

```powershell
git checkout -b test/summarizer-check
Add-Content README.md "`n<!-- summarizer test -->"
git add README.md && git commit -m "test: trigger summarizer"
git push origin test/summarizer-check
gh pr create --title "Test: PR Summarizer" --base workshopBranch
```

After the workflow runs, check the PR for the automated comment.

## What You Built

| Component | Purpose |
|---|---|
| `config.py` | Loads credentials from environment |
| `github_integration.py` | Fetches PR data, posts comments |
| `copilot_analyzer.py` | Generates AI analysis via OpenAI |
| `agent.py` | Orchestrates the full pipeline |
| `copilot-pr-summarizer.yml` | Runs the agent on every PR automatically |
