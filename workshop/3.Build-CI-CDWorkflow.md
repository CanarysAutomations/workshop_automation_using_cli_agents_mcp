# Exercise 3: Build CI/CD Workflows

> **Time:** ~40 minutes
> **Standalone:** Requires Exercise 1 (`helpdesk-crm-agent` must exist).

## Goal

Use Copilot CLI to generate three GitHub Actions workflow YAML files, push them to GitHub, verify CI passes, then introduce a deliberate dependency issue to set up the self-healing exercise.

---

## Steps

**1.** Start Copilot CLI chat:

```powershell
gh copilot chat
```

**2.** Generate the CI workflow:

```
@helpdesk-crm-agent

Generate a GitHub Actions CI workflow (ci.yml) for the Help Desk FastAPI app:
- Trigger: push to any branch, pull_request to main
- Python 3.11, install requirements.txt, cache pip
- Run flake8 (lint) then pytest with coverage
- Upload coverage report as artifact
```

Save the output to `.github/workflows/ci.yml`.

**3.** Generate the CD workflow:

```
@helpdesk-crm-agent

Generate a GitHub Actions CD workflow (cd.yml):
- Trigger: push to main only, after CI passes
- Build Docker image tagged with Git SHA
- Push to GitHub Container Registry (ghcr.io)
- Optional deploy step for Azure App Service (commented out, uses AZURE_WEBAPP_NAME secret)
- Slack notification on success and failure (uses SLACK_WEBHOOK_URL secret)
```

Save the output to `.github/workflows/cd.yml`.

**4.** Generate the issue automation workflow:

```
@helpdesk-crm-agent

Generate a GitHub Actions issue automation workflow (issue-automation.yml):
- Trigger: issue opened or labeled
- Auto-assign based on label: bug→developers, docs→tech-writers, infra→devops
- Add priority label from title keywords: urgent→high, minor→low
- Post an acknowledgment comment
```

Save the output to `.github/workflows/issue-automation.yml`.

**5.** Commit and push all three workflow files:

```powershell
git add .github\workflows\
git commit -m "ci: Add CI/CD and issue automation workflows"
git push origin $(git branch --show-current)
```

**6.** Watch the CI run:

```powershell
gh run list --limit 3
gh run watch $(gh run list --limit 1 --json databaseId -q '.[0].databaseId')
```

Confirm CI shows ✓ before continuing.

**7.** Introduce a breaking dependency to trigger self-healing in Exercise 4.

In `app/main.py`, add at the top (after existing imports):

```python
import requests  # Exercise 4: self-healing demo
```

Uninstall the package and push:

```powershell
pip uninstall requests -y
git add app\main.py
git commit -m "demo: trigger self-healing workflow"
git push origin $(git branch --show-current)
```

**8.** After ~2 minutes, confirm CI now fails:

```powershell
gh run list --limit 3
```
