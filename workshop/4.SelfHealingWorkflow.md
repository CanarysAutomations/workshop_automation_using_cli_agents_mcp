# Exercise 4: Self-Healing Workflow In Action

> **Time:** ~30 minutes
> **Standalone:** Requires Exercise 3 (CI must be failing with a missing `requests` dependency).

## Goal

Observe the self-healing workflow trigger automatically after the CI failure, use Copilot CLI to inspect the fix, verify it locally, and merge the automated PR.

---

## Steps

**1.** Start Copilot CLI chat from the project root:

```powershell
gh copilot chat
```

**2.** Find the self-healing workflow run:

```
Show me the recent workflow runs and identify which self-healing workflow triggered after the CI failure.
```

Wait 2–3 minutes from the failed push if nothing appears yet.

**3.** Analyse what was fixed:

```
Analyze the self-healing workflow logs and explain what dependency was detected and how it was fixed.
```

Expected summary: `requests` was identified as missing, its latest stable version was added to `requirements.txt`, and all tests were re-run and passed.

**4.** Find the self-healing PR:

```powershell
gh pr list --label self-healing
```

Note the PR number.

**5.** Open it in the browser:

```powershell
gh pr view <pr-number> --web
```

Verify: authored by `github-actions[bot]`, diff shows `requests>=X.X.X` in `requirements.txt`, all CI checks are green.

**6.** Checkout the branch and test the fix locally:

```powershell
gh pr checkout <pr-number>
pip install -r requirements.txt
python -c "import requests; print('OK')"
pytest tests/ -v
```

All tests should pass.

**7.** Merge with squash:

```powershell
gh pr merge <pr-number> --squash --delete-branch
```

**8.** Confirm CI on your feature branch is now passing:

```powershell
gh run list --limit 3
```

---

## What Happened

| Stage | Action |
|---|---|
| CI failed | `ModuleNotFoundError: No module named 'requests'` |
| Self-healing triggered | Workflow parsed logs, identified package, added to `requirements.txt` |
| Self-healing PR created | `github-actions[bot]` opened `self-healing/auto-fix-*` |
| Verified & merged | All tests pass, CI is green |
